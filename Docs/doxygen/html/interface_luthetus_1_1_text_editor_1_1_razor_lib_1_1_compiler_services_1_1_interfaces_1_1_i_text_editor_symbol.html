SymbolKindString is not an Enum here.

This is because ITextEditorSymbol is a class within the Text Editor library; every enum would have to be specified by the library itself.

So, (2023-07-20) this hacky string based SymbolKindString is going to be used for now. This allows the consumer of the Text Editor library to add further SymbolKind(s) of their choosing.
