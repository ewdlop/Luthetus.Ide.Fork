Detailed Description

Used when invoking a function.

I'm going to experiment with making this a IExpressionNode. Because the parameters are exclusive to the expression parsing logic, and having to wrap this in a 'BadExpressionNode' when dealing with expressions is very hard to read. (2024-10-26)

TODO: I don't like how this type has the 'OpenParenthesisToken, and CloseParenthesisToken'... ...It was done this way in order to mirror the generic parameters. | Because for generic parameters, the 'OpenAngleBracketToken, and CloseAngleBracketToken' are tied to the existance of generic parameters (i.e.: you must match at least 1 generic parameter if the 'OpenAngleBracketToken' is there.). | With the function parameters however, the 'OpenParenthesisToken' does not mandate that at least 1 function parameter must be matched.
