Event Documentation

◆ ProgressChanged

Action<bool>? Luthetus.Common.RazorLib.Reactives.Models.ProgressBarModel.ProgressChanged

When SetProgress(double) is invoked, then this event is raised with the bool value of 'false'. In this scenario the DecimalPercentProgress is "implied" to have changed, and perhaps one should re-render the progress bar UI.

When Dispose() is invoked, then this event is raised with the bool value of 'true'. In this scenario, the subscriber should unsubscribe from this event, as SetProgress(double) is no longer allowed to be invoked. Note: if one has a reference to this object, they can still invoke GetProgress(), even if the IsDisposed property is set to a value of 'true'.
