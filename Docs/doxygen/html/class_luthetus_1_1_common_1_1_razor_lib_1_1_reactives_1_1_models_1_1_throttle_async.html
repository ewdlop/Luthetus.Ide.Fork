Detailed Description

ThrottleAsync does not achieve the desired result. It blocks itself, and therefore is useless? I like the idea I had with a blocking background task. But it doesn't really extend here much, at least given how its currently written?

Okay, I see, ThrottleAsync works SOLELY from the idea that you are using a fire and forget Task.Run and want to throttle the logic within the Task.Run at a top level. But you cannot use it inside a foreach loop because you are awaiting yourself.

In short ThrottleAsync is actually useful... because it keeps two invocations of code running concurrently. Because one has to wait for the other to finish.

And that many Task.Run will cancel out to the most recent one. But it still seems quite asinine?
