\doxysubsection{Detailed Description}
Sometimes when using the \doxylink{class_luthetus_1_1_common_1_1_razor_lib_1_1_reactives_1_1_models_1_1_throttle}{Throttle}, one is just re-\/using the same Func. 

The current implementation of \doxylink{class_luthetus_1_1_common_1_1_razor_lib_1_1_reactives_1_1_models_1_1_throttle}{Throttle} allows one to \textquotesingle{}Run\textquotesingle{} various Func(s) all through the same throttle.

And this has its uses. But, if you only want to run 1 Func each \textquotesingle{}Run\textquotesingle{} invocation, why re-\/allocate a Func object and closure the variables? \texorpdfstring{$\vert$}{|} Followup\+: one can use a conditional branch within the singular Func to replicate this logic. Add a property to the TArgs that is a bool, enum, or etc... and then if or switch on it.

So, this verson will take in the constructor, the Func to \textquotesingle{}Run\textquotesingle{}.

Then, if any closure is necessary, use the generic argument to this type to define the args.

Preferably, have the args be a struct, and then a pretty sizable optimization (relative to garbage collection CPU usage and memory) is expected.

Side note\+: why does \doxylink{class_luthetus_1_1_common_1_1_razor_lib_1_1_reactives_1_1_models_1_1_throttle}{Throttle} use a Stack for the work items, just to take the top work item, then clear the stack within a lock? \texorpdfstring{$\vert$}{|} Isn\textquotesingle{}t it equivalent to just set a field from within a lock which is simply a \textquotesingle{}Func$<$\+Cancellation\+Token, Task$>$\textquotesingle{} and avoid the stack entirely?
