\doxysubsection{Detailed Description}
Stores the \doxylink{struct_luthetus_1_1_text_editor_1_1_razor_lib_1_1_characters_1_1_models_1_1_rich_character}{Rich\+Character} class instances that represent the text.~\newline
~\newline
 Each \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_text_editors_1_1_models_1_1_text_editor_model}{Text\+Editor\+Model} has a unique underlying resource uri.~\newline
~\newline
 Therefore, if one has a text file named "{}my\+Homework.\+txt"{}, then only one \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_text_editors_1_1_models_1_1_text_editor_model}{Text\+Editor\+Model} can exist with the resource uri of "{}my\+Homework.\+txt"{}.~\newline
~\newline
. 

In regards to the use of the words\+: "{}line"{}, and "{}row"{}\+: A "{}line"{} is being defined as a contiguous block of characters, up to and including a \doxylink{struct_luthetus_1_1_text_editor_1_1_razor_lib_1_1_rows_1_1_models_1_1_line_end}{Line\+End}.~\newline


A "{}row"{} is being defined as a UI rendering term. Such that, without a line-\/wrap, a line of text will be rendered at the same \textquotesingle{}y-\/axis\textquotesingle{} position, with only a difference in \textquotesingle{}x-\/axis\textquotesingle{} between each character. With line-\/wrap, a line can span many "{}row"{}(s) on the UI. That is, the same line of text can be rendered as \textquotesingle{}1 row\textquotesingle{} up until the \textquotesingle{}x-\/coordinate\textquotesingle{} goes out of view. At that point the \textquotesingle{}y-\/axis\textquotesingle{} will be incremented by the height of 1 \textquotesingle{}line\textquotesingle{}, and the \textquotesingle{}x-\/coordinate\textquotesingle{} is reset, and more of the line can be rendered.~\newline
~\newline

Throughout this library, one finds Key$<$\+T$>$ to be a unique identifier. However, since \doxylink{namespace_luthetus_1_1_text_editor_1_1_razor_lib_1_1_lexers_1_1_models_a26a1b63ad28ec7955b30bc6c91571eca}{Resource\+Uri} should be unique, \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_text_editors_1_1_models_1_1_text_editor_model}{Text\+Editor\+Model} is an exception to this pattern.

The motivation for this change comes from calculating the virtualization result.

The storage for the underlying data of a \textquotesingle{}Immutable\+List\textquotesingle{} is believed to be more of a \textquotesingle{}tree\textquotesingle{} structure than a contiguous array.

And therefore, when we virtualize vertically, and then horizontally after, this is an incredible amount of overhead if performed on a \textquotesingle{}tree\textquotesingle{}-\/like structure.

A contiguous array is expected to be a dramatic improvement in performance when calculating the virtualization result.

A side note on this change\+: could .NET internally more easily leverage caching with this now being a contiguous array, rather than a \textquotesingle{}tree\textquotesingle{}?
