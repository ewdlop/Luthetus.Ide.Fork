\begin{DoxyItemize}
\item Fix\+: deletion of lines will now scroll by the amount of lines deleted. Previously, this was breaking the virtualization result until one triggered a re-\/calculation.
\item Fix\+: Keybinds first try to match on a Java\+Script \textquotesingle{}event.\+key\textquotesingle{} so to speak. Then, as a fallback they will now try to match on \textquotesingle{}event.\+code\textquotesingle{} so to speak. Previously, on Ubuntu, if one remapped the Caps\+Lock key to Escape, it would not work in the IDE at various places. This has been fixed.
\item Text editor events now use structs to transmit event data. This is expected to be a large optimization, as it tends that high turnover \textquotesingle{}class\textquotesingle{} objects bring performance issues due to the garbage collection overhead.
\item Text editor\textquotesingle{}s On\+Key\+Down\+Late\+Batching event uses a fixed size array for batching events, rather than what previously was a List$<$\+T$>$. This is expected to be a large optimization, as it tends that high turnover \textquotesingle{}class\textquotesingle{} objects bring performance issues due to the garbage collection overhead. As well, it tends to be the case that no more than 3 or 4 keyboard events ever get batched together. So the fixed size array is \textquotesingle{}8\textquotesingle{} keyboard events can be made into a single batch.
