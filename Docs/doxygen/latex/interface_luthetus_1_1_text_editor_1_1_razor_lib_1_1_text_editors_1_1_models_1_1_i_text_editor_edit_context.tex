If one invokes, "{}\+\_\+ = edit\+Context.\+Get\+View\+Model\+Modifier(view\+Model\+Key);"{}.~\newline
 Then a side effect occurs where the view model with the provided view\+Model\+Key, will be marked as a having been modified.~\newline
 After a Post is finished, the text editor service iterates over all \textquotesingle{}modified\textquotesingle{} view models and re-\/renders them.~\newline
 This behavior is a bit hacky since it requires some amount of \textquotesingle{}hidden knowledge\textquotesingle{}.~\newline
 To get a view model without marking it as modified, use the optional parameter, \textquotesingle{}is\+Readonly\textquotesingle{} and set it to \textquotesingle{}true\textquotesingle{}.~\newline
 The main reason for making this comment, is that I sometimes assign the result of \textquotesingle{}Get\+View\+Model\+Modifier\textquotesingle{} to discard.~\newline
 When I do this it feels especially hacky, so I want to leave a paper trail of what I mean by that statement.~\newline
.
