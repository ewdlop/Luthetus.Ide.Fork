\doxysubsection{Detailed Description}
While parsing expression, it is necessary that there exists a node that indicates a lambda expression is being parsed. 

This type might be more along the lines of a "{}builder"{} type. It is meant to be made when starting lambda expression, then the primary expression can be equal to an instae of this type.

This then directs the parser accordingly until the lambda expression is fully parsed.

At this point, it is planned that a \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_function_definition_node}{Function\+Definition\+Node} will be made, and a \textquotesingle{}Method\+Group\+Expression\+Node\textquotesingle{} (this type does not yet exist) will be returned as the primary expression.

When parsing an expression, there may be a \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_function_invocation_node}{Function\+Invocation\+Node} with a \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_generic_parameters_listing_node}{Generic\+Parameters\+Listing\+Node}. 

The expression code works however by passing around an \textquotesingle{}IExpression expression\+Primary\textquotesingle{} and an \textquotesingle{}IExpression expression\+Seconday\textquotesingle{}.

This means we cannot add common logic for parsing a \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_generic_parameters_listing_node}{Generic\+Parameters\+Listing\+Node} unless the expression parsing code were modified, or maybe a call to a separate function to parse \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_generic_parameters_listing_node}{Generic\+Parameters\+Listing\+Node} could be done.

But, for now an experiment with making \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_generic_parameters_listing_node}{Generic\+Parameters\+Listing\+Node} into a IExpression\+Node is being tried out.

Any IExpression\+Node that is part of a complete expression, will have the IExpression\+Node.\+Result\+Type\+Clause\+Node of Pseudo in order to indicate that the node is part of a expression, but not itself an expression.

Use \textquotesingle{}\doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_type_clause_node}{Type\+Clause\+Node} IExpression\+Node.\+Result\+Type\+Clause\+Node =\texorpdfstring{$>$}{>} Type\+Clause\+Node\+Facts.\+Pseudo;\textquotesingle{} so that some confusion can be avoided since one has to cast it explicitly as an IExpression\+Node in order to access the property.
