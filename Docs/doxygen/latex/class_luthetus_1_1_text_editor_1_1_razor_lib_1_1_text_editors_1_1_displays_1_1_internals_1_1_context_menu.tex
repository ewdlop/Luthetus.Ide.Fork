When implementing this type, add the Blazor attribute \textquotesingle{}\mbox{[}Parameter\mbox{]}\textquotesingle{}, and the text editor will provide its instance to the dependent component via this Blazor parameter. 

The only way for a dependent component to re-\/render is via the event\+: \doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_text_editors_1_1_displays_1_1_text_editor_view_model_display_adadf9a7691d195fafa0cad6e83047e41}{Text\+Editor\+View\+Model\+Display.\+Render\+Batch\+Changed} (i.\+e.\+: the \textquotesingle{}State\+Has\+Changed()\textquotesingle{} for the text editor will not cascade to the dependent component).

One can subscribe to this event and throttle re-\/rendering as desired. This allows the accessory UI that surrounds the text editor to be rendered at a "{}lower priority"{} than the text editor itself.
