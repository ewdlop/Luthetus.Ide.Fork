Some terminal commands will map to "{}interactive"{} commands that are meant to modify the shell session rather than to start a process. 

Ex\+: \textquotesingle{}cd ../..\textquotesingle{} is an "{}interactive"{} command, where as \textquotesingle{}dotnet run\textquotesingle{} will start a process.

This method gives the \doxylink{interface_luthetus_1_1_ide_1_1_razor_lib_1_1_terminals_1_1_models_1_1_i_terminal_interactive}{ITerminal\+Interactive} an opportunity to handle the command, before it is executed at a more general level.

Presumably, \textquotesingle{}dotnet run ../\+My\+Project.csproj\textquotesingle{} where there is a "{}.."{} in a path, would need to be taken by the \doxylink{interface_luthetus_1_1_ide_1_1_razor_lib_1_1_terminals_1_1_models_1_1_i_terminal_interactive}{ITerminal\+Interactive}, and then modified to replace "{}../\+My\+Project.\+csproj"{} with the result of traversing this relative path from the working directory path.

The final result would then be returned for the \doxylink{interface_luthetus_1_1_ide_1_1_razor_lib_1_1_terminals_1_1_models_1_1_i_terminal}{ITerminal} to execute, "{}dotnet run C\+:/\+User/\+My\+Project/\+My\+Project.\+cs"{} if the working directory were to be "{}\+C\+:/\+User/\+My\+Project/wwwroot/"{}.

If null is returned, then the \doxylink{interface_luthetus_1_1_ide_1_1_razor_lib_1_1_terminals_1_1_models_1_1_i_terminal}{ITerminal} should return (do nothing more).
