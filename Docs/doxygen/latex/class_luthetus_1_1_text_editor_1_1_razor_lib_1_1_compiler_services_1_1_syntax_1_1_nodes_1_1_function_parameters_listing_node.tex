\doxysubsection{Detailed Description}
Used when invoking a function. 

I\textquotesingle{}m going to experiment with making this a \doxylink{interface_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_97d7489c2c3077ac339153787f61d38f}{IExpression\+Node}. Because the parameters are exclusive to the expression parsing logic, and having to wrap this in a \textquotesingle{}\doxylink{class_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_nodes_1_1_bad_expression_node}{Bad\+Expression\+Node}\textquotesingle{} when dealing with expressions is very hard to read. (2024-\/10-\/26)

TODO\+: I don\textquotesingle{}t like how this type has the \textquotesingle{}Open\+Parenthesis\+Token, and Close\+Parenthesis\+Token\textquotesingle{}... ...It was done this way in order to mirror the generic parameters. \texorpdfstring{$\vert$}{|} Because for generic parameters, the \textquotesingle{}\doxylink{struct_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_tokens_1_1_open_angle_bracket_token}{Open\+Angle\+Bracket\+Token}, and \doxylink{struct_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_tokens_1_1_close_angle_bracket_token}{Close\+Angle\+Bracket\+Token}\textquotesingle{} are tied to the existance of generic parameters (i.\+e.\+: you must match at least 1 generic parameter if the \textquotesingle{}\doxylink{struct_luthetus_1_1_text_editor_1_1_razor_lib_1_1_compiler_services_1_1_syntax_1_1_tokens_1_1_open_angle_bracket_token}{Open\+Angle\+Bracket\+Token}\textquotesingle{} is there.). \texorpdfstring{$\vert$}{|} With the function parameters however, the \textquotesingle{}Open\+Parenthesis\+Token\textquotesingle{} does not mandate that at least 1 function parameter must be matched.
