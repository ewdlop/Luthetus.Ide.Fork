\doxysubsection{Detailed Description}
It is preferred that each test defines its own test data within the \textquotesingle{}\mbox{[}Fact\mbox{]}\textquotesingle{} method itself. There however is 100 lines of code at the beginning of every test if one follows this rule. To reduce the noise, this method was created.~\newline
~\newline
. 

That being said, it is vital that the test defines its test-\/data, otherwise some readonly and static data might one day have the source code for its definition change. Any dependent tests would then break, and a great deal of confusion would arise, because it would not be clear that the test data was changed.~\newline
~\newline


For this reason, the \textquotesingle{}\mbox{[}Fact\mbox{]}\textquotesingle{} method must invoke this method with every parameter needed to construct the test data. In this method, the exact same parameters that were passed in will be typed out in the source code for this method.~\newline
~\newline


If the parameters to this method, and the hardcoded local variables of this method match, then the test and this method agree on what the test data is. At that point the test data can be constructed, and the pre-\/assertions can be done from within this method.~\newline
~\newline


Lastly, return the test data.
